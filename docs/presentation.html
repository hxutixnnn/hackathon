The Problem

Auditing a repo for vulns is slow, shallow, or both

Sequential scanners

Snyk / Semgrep / GHAS pattern-match. Miss logic bugs. Ignore intent.

Single-LLM scanners

One prompt, one pass. Token limit kills coverage past ~50 findings.

"Just use Codex CLI"

Nondeterministic. No cache. No benchmark. No way to prove accuracy.

15 attack angles × 10 file chunks × 4s sequential = 10 minutes per scan. Demo unusable.

The Solution

Paste GitHub URL → 15 auditors fan out per file chunk → findings stream live

  • 15 attack angles per chunk: SQLi, XSS, authz, SSRF, secrets, race, …
  • Workpool concurrency 20 — bounded parallelism, retry
  • Constellation UI — node per auditor, pulses on new finding
  • Convex live queries — zero polling, real-time stream

90s vs 10 min sequential

Architecture

Convex orchestrates · Workpool bounds · OpenAI scores · Browser subscribes


Browser ──WS── Convex ──schedule──> orchestrator (Node)
                  │                      │
                  │                      └─enqueue × N──> Workpool(20) ──> agent ──> OpenAI
                  │                                                          │
                  │                                                          ├─ cache.lookup (sha256 key)
                  │                                                          │     hit  → reuse findings
                  │                                                          │     miss → call OpenAI, cache.put
                  │                                                          │
                  │                                                          └─ findings.insertMany
                  │                                                                    │
                  │      (last agent fires) ───> dedup.run                              │
                  │                                ├─ embed (text-embedding-3-small)   │
                  │                                ├─ union-find cluster (cosine ≥0.85)│
                  │                                └─ LLM rerank top-30                │
                  │                                                                    │
                  │      (juice-shop URL?) ───> eval.score → benchmarks (P/R/F1) ──────┘
                  │
                  └── live queries ── React (constellation, table, drawer)

No polling. No queues we run. Convex scheduler + Workpool do it all.

The Accuracy Moat

Three shipped systems a "Codex CLI + prompt engineer" cannot replicate over a weekend

Content-addressed cache

sha256(angle|model|promptVer|chunk)

Re-scan unchanged code = 0 LLM tokens. Hit/miss live in StatsBar. PROMPT_VER bump invalidates.

Embedding + union-find dedup

Stage A: deterministic clustering — cosine ≥ 0.85, gated by file + line overlap. Pure math.

Stage B: LLM rerank top-30 only. Single-LLM reducers cap ~50; we scale past 1000.

Juice-shop F1 benchmark

~20 hand-curated truth rows from owasp-juice.shop. Every juice-shop scan scored.

Live P · R · F1 in BenchmarkPanel + green-dot matched-truth column.

Race-free: dedupStartedAt guard inside bumpProgress — atomic schedule, no double-fire.
